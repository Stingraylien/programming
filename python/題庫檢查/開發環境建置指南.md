# 開發環境建置指南

本文件旨在協助開發人員快速建置開發環境、了解專案相依性，以及如何將 Python 腳本打包為獨立執行檔。

## 系統需求
- Python 3.8 或以上版本
- 內建 `tkinter` 模組
- 作業系統：Windows 或 macOS

## 環境建置步驟

### 1. 建立並啟動虛擬環境
```bash
# 建立虛擬環境
python -m venv .venv

# 啟動虛擬環境 (macOS / Linux)
source .venv/bin/activate

# 啟動虛擬環境 (Windows Command Prompt)
.venv\Scripts\activate
```

### 2. 安裝相依套件
本專案依賴 Pandas 進行表單資料操作，並需要 xlsxwriter 作為產生報告的引擎。
```bash
pip install pandas openpyxl xlsxwriter
```

## 專案架構與核心程式
- 主程式為 `Check.py`。
- 其他腳本如 `Check_wins.py` 與 `Check_MacOS.py` 為針對不同環境預留的特化版本，日常維護請以 `Check.py` 為準。
- **資料分群邏輯**：程式利用 DataFrame 的 `ffill()` 向下填補空值，並使用 `groupby("Subject")` 將同題目的選項群組化以進行邏輯驗證。

## 獨立執行檔打包 (PyInstaller)
使用 PyInstaller 可以將程式打包為單一執行檔，方便給無 Python 環境的使用者。

```bash
pip install pyinstaller

# Windows 或 Mac 皆可使用的通用打包指令 (-F 單一檔案, -w 隱藏終端機視窗)
pyinstaller -F -w Check.py
```
打包完成後，執行檔會生成在 `dist/` 資料夾中。
若專案目錄中已有 `.spec` 檔案 (例如 `Check.spec`)，也可以直接執行 `pyinstaller Check.spec` 來打包。
