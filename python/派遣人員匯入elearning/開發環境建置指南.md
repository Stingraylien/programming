# 開發環境建置指南

本文件旨在協助開發人員快速建置開發環境、了解專案相依性，以及如何將 Python 腳本打包為獨立執行檔。

## 系統需求
- Python 3.8 或以上版本
- 作業系統：Windows 或 macOS

## 環境建置步驟

### 1. 建立並啟動虛擬環境 (強烈建議)
為了避免套件衝突，建議在專案目錄下建立獨立的虛擬環境：
```bash
# 建立虛擬環境
python -m venv .venv

# 啟動虛擬環境 (macOS / Linux)
source .venv/bin/activate

# 啟動虛擬環境 (Windows Command Prompt)
.venv\Scripts\activate
```

### 2. 安裝相依套件
本專案使用 tkinter 繪製 GUI (Python 內建)，並依賴 Pandas / Openpyxl 來處理 Excel 資料的讀取與轉換。
```bash
pip install pandas openpyxl
```

## 專案開發說明
- **核心邏輯**：資料欄位的對應邏輯以及 I/O 規則，請詳閱同目錄下的 `程式說明.md`。
- **UI 介面**：使用內建 `tkinter.filedialog` 進行檔案選取。

## 獨立執行檔打包打包 (PyInstaller)
若需提供給無 Python 環境的終端使用者（如 HR 人員），可以使用 PyInstaller 將程式打包為單一執行檔。

### 安裝 PyInstaller
```bash
pip install pyinstaller
```

### 執行打包指令
使用 `-F` 參數打包成單一檔案，`-w` 參數隱藏終端機背景視窗（適合 GUI 程式）。

**Windows 環境打包**：
```bash
pyinstaller -F -w windows_派遣人員匯入elearning.py
```
*或是使用通用的程式：*
```bash
pyinstaller -F -w 通用_派遣人員匯入elearning.py
```

打包完成後，獨立的執行檔 (`.exe` 或 Mac 的 `.app` / Unix 執行檔) 會生成在 `dist/` 資料夾中。請將該檔案交予使用者即可。
