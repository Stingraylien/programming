# 開發環境建置指南 (Developer Guide)

本文件紀錄了如何從零開始建立「PDF to Markdown Converter」的開發與執行環境。若您是接手開發或是需要重新部署，請遵循以下步驟。

## 1. 系統需求
- **作業系統**：macOS / Windows / Linux 均可支援。
- **Python 版本**：建議使用 Python 3.10 或以上版本（本專案於 Python 3.12 測試通過）。

## 2. 環境建置流程

### 建立虛擬環境 (建議)
為了避免套件衝突，建議您在專案根目錄下建立虛擬環境：
```bash
python -m venv venv
source venv/bin/activate  # macOS / Linux
# 或者是 venv\Scripts\activate  # Windows
```

### 安裝必要相依套件 (Dependencies)
本專案依賴以下第三方套件：
- `PySide6`：用於建立現代化的圖形化使用者介面 (GUI)。
- `pymupdf4llm`：核心的 PDF 解析與轉換引擎。

請在終端機中執行以下指令進行安裝：
```bash
pip install PySide6 pymupdf4llm
```

## 3. 程式碼架構簡介
- 本專案採用物件導向設計 (`class PDFToMarkdownApp`)。
- 為確保 GUI 介面不會因為處理大量 I/O (檔案讀寫) 而發生凍結 (Freeze)，核心轉換邏輯 `process_folder` 是在獨立的 Background Thread 中執行的。
- 跨執行緒的畫面更新（如印出日誌、更新進度條）是嚴格透過 Qt 的 `Signal` 機制 (`WorkerSignals`) 來安全進行的。

## 4. 打包為獨立執行檔 (未來擴充)
如果您需要將此程式提供給完全沒有安裝 Python 的使用者，您可以使用 `PyInstaller` 將其封裝。

**安裝 PyInstaller:**
```bash
pip install pyinstaller
```

**打包指令:**
```bash
pyinstaller --noconsole --onefile pdf_to_md_converter.py
```
這會在 `dist/` 資料夾下產生一個可以直接雙擊執行的執行檔（在 Mac 上為 UNIX 執行檔，在 Windows 上為 `.exe`）。
